# Documentation

Documentation is organized by audience and authority:

- [`reference/bibliotek/`](reference/bibliotek/) contains generated human views of the Bibliotek
  library, its components, shared semantics, and dependency topology.
- [`reference/vellis/`](reference/vellis/) contains the generated Vellis application composition,
  use cases, façade, verification, and realization mappings.
- [`engineering/sysml-modeling.md`](engineering/sysml-modeling.md) explains the repository's SysML
  profile, model workflow, validation, and packaging rules.
- [`guides/vellis/evals/`](guides/vellis/evals/) contains Vellis evaluation prompts, walkthroughs,
  and operating guidance.
- [`design/open-design-questions.md`](design/open-design-questions.md) is a non-normative backlog of
  unresolved model-design questions.
- [`vision/agentic-mbse-engineering-system.md`](vision/agentic-mbse-engineering-system.md) describes
  the longer-term model-based engineering thesis.
- [`migration/`](migration/) contains temporary evidence retained only for the SysML cutover.

Textual SysML under `../model/` is the authored design. Files under `reference/` are generated by
`just model-render` and must not be edited as alternate specifications. Machine-consumable generated
indexes and conformance data live under `../generated/model/`, outside the human documentation tree.
