8.4.14.1  State Definitions

A StateDefinition       is a kind ofActionDefinition. As such, all the general semantic constraints for an
ActionDefinition       (see 8.4.13.1 ) also apply to aStateDefinition. In addition, the
checkStateDefinitionSpecialization              constraint requires that aStateDefinition       specialize the base
StateDefinition      States::StateAction         (see 9.2.11.2.1 ), which specializes the ActionDefinition
Actions::Action      (see 9.2.10.2.4 ) and the kernel Behavior    StatePerformances::StatePerformance               (see
[KerML, 9.2.6]).

       state def     S  specializes      States::StateAction;
       state    def  S1   specializes      S;

Further, thecheckActionUsageStateActionRedefinition                 constraint (see 8.4.13.2 ) requires that an
ActionUsage     that is anentryAction,     exitAction    or doAction    of a StateDefinition      (i.e., is owned via a
StateSubactionMembership) redefine, respectively, the         entryAction,     exitAction, or     doAction   feature
ofStateAction     (see 9.2.11.2.1 ), which redefine the  entry,  exit,  and do  features    of
StatePerformances::StatePerformance              (see [KerML, 9.2.6]), restricting them to be   Actions   (see
9.2.10.2.4 ).
